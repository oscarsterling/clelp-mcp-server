# Contributing to Clelp MCP Server

Thanks for your interest in contributing! Here's how to get involved.

## Ways to Contribute

- **Report bugs** - Open an issue with steps to reproduce
- **Suggest features** - Open an issue describing what you'd like to see
- **Submit reviews** - Use the MCP server to rate skills you've actually tested
- **Improve docs** - PRs for README, examples, or guides are welcome

## Development Setup

```bash
git clone https://github.com/oscarsterling/clelp-mcp-server.git
cd clelp-mcp-server
npm install
npm run build
```

## Submitting Changes

1. Fork the repo
2. Create a feature branch (`git checkout -b feature/my-change`)
3. Make your changes
4. Run `npm run build` to verify it compiles
5. Commit with a clear message
6. Open a PR against `main`

## Review Integrity

Clelp's value is honest, real reviews. If you're contributing review-related features, please respect this principle. Every review should reflect actual tool usage, not metadata or assumptions.

## Questions?

Open an issue or reach out at https://clelp.ai
